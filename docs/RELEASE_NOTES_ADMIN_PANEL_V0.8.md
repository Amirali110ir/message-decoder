## خلاصه اجرایی (برای تیم محصول)

پنل ادمین تا پیش از این یک داشبورد نظارتی ساده بود — اعداد بالا، جدول‌های بدون صفحه‌بندی، و چهار ابزار مهم کیفیت (یادگیری روزانه، ارزیابی موتور قانون، کیس‌های فیدبک منفی، و تست زنده) که در بک‌اند آماده بودند ولی هیچ دسترسی UI‌ای به آن‌ها وجود نداشت.

این نسخه سه نقص اصلی را برطرف می‌کند:

۱. **شکاف ابزار کیفیت:** پنل‌های یادگیری و موتور قانون به داشبورد اضافه شدند — تیم محصول می‌تواند بدون SQL یا terminal دقت مدل را ببیند، کیس‌های فیدبک منفی را بررسی کند، و هر پیامی را زنده تست کند.
۲. **ضعف عملیاتی:** کاربران تلگرامی بدون شماره قابل مدیریت اعتبار نبودند و جدول‌ها بدون صفحه‌بندی با رشد کاربران کند می‌شدند.
۳. **کمبود داده‌محور:** خروجی CSV برای همه جدول‌ها و رفرش خودکار برای داشبورد اضافه شد.

**اثر تجاری:** تیم می‌تواند کیفیت مدل را هر روز پیش از ارسال اعتبار یا broadcast بررسی کند؛ کاهش regret و بهبود retention.

## تصمیم‌های کلیدی و منطقشان

| تصمیم | چرا این رویکرد |
|---|---|
| بخش‌های یادگیری/rule-engine **on-demand** هستند (نه auto-load) | این endpointها محاسباتی‌اند؛ auto-load آن‌ها در هر رفرش داشبورد سرور را اضافه‌بار می‌کرد |
| `Promise.allSettled` برای load داشبورد حفظ شد | failure تلگرام نباید کل پنل را خراب کند |
| `adjustCredits` برای `user_id` تلگرام هم کار می‌کند | کاربران تلگرام اغلب شماره ندارند؛ disable کردن دکمه +/- ابزار مدیریت را ناقص می‌کرد |
| Pager فقط وقتی بیش از یک صفحه داده هست نمایش داده می‌شود | کمتر کردن noise UI برای دیتابیس‌های کم داده (محیط dev) |
| CSV با `﻿` (BOM) صادر می‌شود | Excel در Windows بدون BOM فارسی را garbled نشان می‌دهد |

## تغییرات به تفکیک

### T1 — wrapperهای API یادگیری و موتور قانون
`apps/web/lib/api.ts`
- افزودن `adminLearningDaily`, `adminRuleEngineEval`, `adminRuleEngineCandidates`, `adminRuleEngineExplain` با تایپ‌های کامل
- **اثر:** endpointهای بک‌اند که ۳ ماه بدون UI بودند قابل استفاده شدند

### T2 — پنل یادگیری روزانه
`apps/web/app/admin/page.tsx`
- انتخاب تاریخ (default: دیروز UTC)، نمایش ۹ متریک (decode، paid، copy rate، feedback+/−، regret، lens/safety mix)
- توصیه‌های خودکار بک‌اند به‌صورت لیست

### T3 — پنل کیفیت موتور قانون
`apps/web/app/admin/page.tsx`
- **Eval:** دقت لنز/ایمنی/لحن، confusion matrix، جدول misses
- **Candidate cases:** جدول فیدبک‌های منفی با export CSV
- **Explain:** تست زنده هر پیام — لنز، ایمنی، لحن، playbook

### T4 — صفحه‌بندی جدول‌ها
- کاربران وب، اتفاقات وب (هر صفحه ۵۰ ردیف) با دکمه‌های قبلی/بعدی و شمارنده موقعیت
- جستجوی جدید offset را صفر می‌کند

### T5 — خروجی CSV
- دکمه `CSV` در heading همه جدول‌های اصلی (تحلیل‌ها، کاربران وب، کاربران تلگرام، اتفاقات، کیس‌های کاندید)
- دانلود سمت کلاینت بدون نیاز به endpoint جدید

### T6 — رفع اعتبار تلگرام + رفرش خودکار
- `adjustCredits` با `user_id` تلگرام (برای کاربران بدون شماره)؛ دکمه +/- همیشه فعال
- چک‌باکس رفرش خودکار هر ۳۰ ثانیه

### T7 — بیلد و همگام‌سازی
- `npm run build` → `rsync` به `apps/api/web_static`
- bump نسخه: `0.6.0` → `0.8.0` (skip 0.7.0 که قبلاً tag شده بود)
