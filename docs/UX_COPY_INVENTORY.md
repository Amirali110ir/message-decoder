# UX Copy Inventory - Message Decoder

این فایل مرجع همه متن‌های ثابت سایت است تا UX writer بتواند متن‌ها را صفحه‌به‌صفحه اصلاح کند و نسخه ویرایش‌شده را برای اعمال در کد و دیپلوی برگرداند.

به‌روزرسانی 2026-05-23:
کپی‌های واقعی سایت بر اساس `docs/MARKETING_COPY_GUIDELINES_FROM_TWEETS.md` در فایل‌های `apps/web/app/page.tsx`، `apps/web/app/decoder/page.tsx`، `apps/web/app/admin/page.tsx`، `apps/web/app/layout.tsx` و `apps/web/lib/api.ts` بازنویسی شدند. از این نسخه به بعد، سورس کد منبع دقیق متن فعلی است و این سند بیشتر برای نگه داشتن نقشه URL / محل نمایش / فایل منبع استفاده می‌شود تا دور بعدی UX writer بتواند همان نقاط را ویرایش کند.

روش استفاده:
- ستون/بخش `متن فعلی` را بخوانید و فقط مقدار `متن پیشنهادی UX` را تغییر دهید.
- `ID`، `URL`، `محل نمایش` و `منبع در کد` را تغییر ندهید؛ این‌ها نقشه اعمال متن در کد هستند.
- متن‌های شامل مقدار پویا مثل `{credits}` یا `{phone}` باید همان placeholder را نگه دارند.
- خروجی‌های تحلیلی که توسط API/AI ساخته می‌شوند، در انتهای فایل جدا شده‌اند چون متن ثابت صفحه نیستند.

## API / AI Generated Texts

این متن‌ها ثابتِ صفحه نیستند، ولی روی سایت نمایش داده می‌شوند چون از backend برمی‌گردند. اگر UX writer بخواهد لحن تحلیل‌ها، پاسخ‌های پیشنهادی، warningها، labelهای لنز، یا متن‌های fallback خروجی AI را تغییر دهد، منبع‌های اصلی این‌ها هستند:

- `apps/api/app/services/ai.py`: system prompt، labelهای لنز، متن‌های safety، fallback analysis، reply labels، reply templates، privacy warning، CTA خروجی مدل.
- `apps/api/app/services/rule_engine.py`: متن‌های rule-based برای حالت‌های safety، نیاز پنهان، ریسک، جهت پاسخ، کلمات ممنوع، core moves.
- `apps/api/app/services/learning.py`: متن‌های recommendation داخلی learning/admin.
- `apps/api/app/routers/*.py`: خطاهای API و پیام‌های endpointها، اگر مستقیماً به frontend برسند.

برای تغییر این بخش‌ها بهتر است UX writer یک بخش جدا با عنوان `AI output tone changes` اضافه کند، چون اعمال آن‌ها فقط جایگزینی متن صفحه نیست و ممکن است روی prompt و رفتار مدل اثر بگذارد.
