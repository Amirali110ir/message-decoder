**تاریخ:** 2026-06-03
**دامنه:** فرانت‌اند وب (`apps/web`) — همهٔ صفحات RTL: فرود، decoder، signup، dashboard

**تعداد تسک:** ۲۵ تسک (T1–T25)، همه کامل · build سبز · تست‌شده روی ویوپورت ۳۷۵px

## ۱. خلاصهٔ اجرایی (برای تیم محصول)

**مسئله‌ی کسب‌وکاری:** Message Decoder یک هم‌راهِ گفت‌وگوست که کاربر **وسطِ یک مکالمهٔ احساسی، روی موبایل** بازش می‌کند. اگر محصول حسِ بومیِ موبایل نداشته باشد — متن به لبه بچسبد، دکمه‌ها ریز باشند، ورودی هنگام تایپ zoom بزند، ورود با کدِ پیامک اصطکاک داشته باشد — همان لحظهٔ حساس، **اعتماد و نرخ تبدیل افت می‌کند.** بازخوردِ مستقیمِ ذی‌نفع هم همین بود: «روی موبایل اصلاً حسِ موبایل ندارد، نسخهٔ کوچک‌شدهٔ دسکتاپ است.» این درست بود.

**چه چیزی عوض شد، به زبان ساده:**
- محصول حالا روی موبایل **مثل یک اپ بومی** رفتار می‌کند: فاصلهٔ درست از لبه‌ها، احترام به ناحیهٔ امنِ نمایشگر (notch)، نوارِ اکشنِ چسبان پایین، و **شیتِ پایین‌چسب (bottom-sheet)** برای ورود به‌جای پنجرهٔ وسط.
- **اصطکاکِ مسیرِ پولی‌شدن کم شد**: دکمهٔ اصلی دیگر «مرده/غیرفعال» نیست (به‌جایش راهنمایی می‌دهد)، و کدِ ورودِ پیامکی حالا **خودکار از کیبوردِ iOS/Android پیشنهاد می‌شود** — این مستقیم روی نرخِ تکمیلِ ثبت‌نام اثر دارد.
- **زوم‌نزدنِ iOS**: ورودی‌ها دیگر هنگام تایپ کلِ صفحه را بزرگ نمی‌کنند (یکی از واضح‌ترین نشانه‌های «وب‌سایت، نه اپ»).
- متن، ارقام و اعدادِ داده حالا **کاملاً فارسی و راست‌چین‌خوان** هستند.
- زیرِ پوست، یک **سیستمِ توکنِ طراحی** (فاصله، شعاع، اندازهٔ لمسی، مقیاسِ فونت) نشست تا توسعهٔ بعدی سریع و یکدست بماند.

**چرا مهم است (تأثیرِ تجاری):** هر سه نقطه‌ای که بیشترین اثر را دارد روی **همان مسیری** است که هر کاربرِ پولی باید از آن عبور کند — اولین تحلیل، ساختِ پاسخِ پولی، و ورود با OTP. این‌ها هزینهٔ تبدیل را پیش از هر بهبودِ ظاهری کم می‌کنند.

## ۲. تصمیم‌های کلیدی و منطقشان (decision-making)

| تصمیم | چرا این، نه آن |
|---|---|
| **context-first به‌جای mobile-first** | موبایل را «محدودترین زمینه» گرفتیم که وضوح را تحمیل می‌کند، نه یک سیلوی جدا. یک سیستمِ واحد که در همهٔ اندازه‌ها resolve شود. |
| **اهداف لمسی روی `@media (pointer: coarse)`، نه بر اساس عرضِ صفحه** | کفِ لمسیِ ۴۴px باید به **نوعِ اشاره‌گر (لمسی)** گره بخورد نه پهنای نمایشگر؛ این‌طور دسکتاپ بی‌دلیل بزرگ نمی‌شود و تبلتِ لمسی هم درست می‌شود. |
| **مهاجرتِ توکنِ فونت به‌صورت پیکسل‌به‌پیکسل** | ۸۷ مقدارِ `font-size` به ۱۵ توکن تبدیل شد بدونِ تغییرِ حتی یک پیکسل → **صفر رگرسیون بصری**، ولی سیستم حالا وجود دارد. ریسکِ صفر، ارزشِ ساختاری بالا. |
| **حذفِ band-aidهای سرریز، اما با مدرک** | قاعدهٔ کورِ `body * { max-width }` و `overflow-x:hidden`های تکراری برداشته شد، بعد با **آشکارسازِ سرریزِ افقی** روی هر ۴ صفحه در ۳۷۵px تأیید شد که سرریز صفر است. حدس نزدیم، اندازه گرفتیم. |
| **bottom-sheet فقط با CSS** | ورود روی موبایل به شیتِ پایین‌چسب با دستگیره تبدیل شد بدونِ افزودنِ کتابخانه/JS سنگین — کم‌ریسک و سریع. |
| **مودال: focus-trap واقعی** | افزودنِ Esc + کلیکِ backdrop + چرخشِ Tab داخلِ مودال (نه فقط ظاهر) — قانونِ دسترس‌پذیریِ dialog. |

**موردِ مهمِ honest:** دو مورد (`T3` حذفِ band-aid و `T18` نزدیک‌سازیِ بازخورد) ابتدا عمداً «معوق» شدند تا کورکورانه چیزی نشکند؛ بعد با تست/نسخهٔ مهارشده بسته شدند.

## ۳. تغییرات به تفکیک (گروه‌بندی‌شده)

### الف) حسِ بومیِ موبایل (بیشترین اثرِ ادراکی)
- **T1** gutter لبه‌ها به ۲۰px + احترام به safe-area چپ/راست.
- **T8** مودالِ ورود → **bottom-sheet** با دستگیرهٔ کشیدن و انیمیشن.
- **T9** دکمهٔ اصلیِ تحلیل → **CTA چسبانِ پایین** (sticky action bar) با احترام به safe-area پایین.
- **T13** + **T12** `viewport-fit=cover` و `env(safe-area-inset-*)` برای notch/home-indicator.
- **T10** نوارِ ناوبری به‌جای اسکرولِ افقی، wrap می‌شود (CTA پنهان نمی‌شود).
- **T11** حالتِ `:active` لمسی به‌جای tap-highlightِ حذف‌شده.

### ب) اندازه و خوانایی
- **T4** فونتِ ورودی‌ها ۱۶px → **توقفِ زوم خودکار iOS** (بحرانی).
- **T5** اندازهٔ متنِ بدنه روی موبایل بزرگ‌تر.
- **T6/T7** اهدافِ لمسیِ ۴۴px روی اشاره‌گرِ لمسی (دکمهٔ لحن، mini-action، آیکن‌ها).
- **T14** حذفِ `min-height`های ثابتِ دسکتاپی که زیرِ فرم جعبهٔ خالیِ بلند می‌ساختند.

### ج) اصطکاکِ مسیرِ تبدیل (اثرِ مستقیمِ درآمدی)
- **T16** دکمهٔ اصلی دیگر `disabled` نیست؛ روی کلیکِ خالی فوکوس + پیامِ راهنما (به‌جای دکمهٔ مرده — یافتهٔ کلاسیکِ Baymard).
- **T17** `autoComplete="one-time-code"` + `inputMode` روی OTP و `autoComplete="tel"` → autofillِ کدِ پیامک در ۳ صفحه.
- **T19** قفل‌شدنِ فقط همان دکمهٔ لحنِ در حال پردازش (نه همه).
- **T18** بازخوردِ status/error با `scrollIntoView` + `aria-live` نزدیکِ کنش دیده می‌شود.
- **T20** مودال: Esc + کلیکِ backdrop + focus-trap.

### د) سیستم و کرفت
- **T3** حذفِ اصلاح‌های زخمیِ سرریز (تأییدشده: صفر سرریز در ۳۷۵px).
- **T21** رنگِ لنزها از `var(--cds-lens-*)` (حذفِ hexِ خام).
- **T22** **مقیاسِ فونت توکنیزه شد** (۱۵ توکنِ `--fs-*`، ۸۷ usage) + توکن‌های radius/touch/gutter.
- **T23** هلپرِ `faNum` در `lib/format.ts` → **ارقامِ فارسی** برای همهٔ دادهٔ عددیِ API.
- **T24** آیکنِ SVG به‌جای اموجیِ باتری/سیگنال در موکاپ.
- **T25** trust-strip با دو ادعای متمایز (به‌جای تکرار).

## ۴. تأیید و معیارِ سنجش

- **چشمی روی ۳۷۵px** تأیید شد: gutter، CTA چسبان، bottom-sheet با دستگیره.

- نرخِ رسیدن از تحلیلِ رایگان به ساختِ پاسخِ پولی (اثرِ مسیرِ گروه ج).
